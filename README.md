# ExoHabitNet — Project Overview and Usage

This repository contains the ExoHabitNet project: a PyTorch-based classification pipeline for identifying exoplanet transit signals and classifying candidates into HABITABLE, NON_HABITABLE, and FALSE_POSITIVE classes. The repo includes preprocessing, training (single-run + K‑fold), OOF aggregation, calibration, evaluation, visualization, and helper scripts for creating ensemble checkpoints.

This README is a focused, practical guide to reproduce results, run experiments, and prepare the repository for handoff.

**Work directory:** Most scripts expect to be run from the `exohabitnet/exohabitnet` subfolder. From the repository root run:

```powershell
cd exohabitnet\exohabitnet
```

## Environment

- Python 3.8+ recommended. GPU highly recommended for training.
- Create and activate a virtual environment, then install dependencies:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1   # PowerShell
pip install -r requirements.txt
```

(If you prefer to run from the repo root, prefix commands with `cd exohabitnet\exohabitnet`.)

## Data layout

- Raw fits: `exohabitnet/data/raw_fits/` (large; intentionally ignored)
- Processed datasets (generated by preprocessing): `exohabitnet/data/processed/` and the CSVs at `exohabitnet/data/processed_dataset.csv`, `exohabitnet/data/train_dataset.csv`, `exohabitnet/data/test_dataset.csv`

Note: Large files (raw FITS, full reports, checkpoints) are ignored by default. The canonical handoff artifacts have been explicitly un-ignored for pushing (see `.gitignore`).

## Typical workflow (reproduce results)

1. Preprocess (start from collected raw files):

```powershell
cd exohabitnet\exohabitnet
python -u scripts/preprocessing_pipeline.py
```

2. (Optional) Pretrain encoder (short run):

```powershell
python -u scripts/train.py --pretrain --pretrain-epochs 10
```

3. Train (single run) — quick validation:

```powershell
python -u scripts/train.py --epochs 30 --loss focal --focal-gamma 2.0 --hab-weight-multiplier 3.0 --use-weighted-sampler --sampler-scale 1.5 --batch-size 32
```

4. K‑Fold training (recommended for final runs):

```powershell
python -u scripts/kfold_finetune.py --loss focal --focal-gamma 2.0 --hab-weight-multiplier 3.0 --epochs 30 --n-splits 5 --use-weighted-sampler --sampler-scale 1.5 --batch-size 32
```

5. Generate OOF probabilities and recommended threshold:

```powershell
python -u scripts/oof_aggregator.py --n-splits 5
```

6. Fit calibrators (Platt + Isotonic) and compare:

```powershell
python -u scripts/oof_calibrate.py
```

7. Apply the chosen calibrator to the holdout test set:

```powershell
python -u scripts/apply_calibrator_to_test.py
```

8. Final evaluation (writes `classification_report.md` and `model_performance.json`):

```powershell
python -u scripts/evaluate.py
```

9. Create ensemble checkpoint (if desired):

```powershell
python -u scripts/create_ensemble_checkpoint.py
# or use the alternate helper
python -u scripts/ensemble_checkpoints.py
```

## Outputs and what they mean

- `exohabitnet/models/checkpoints/kfold/fold_{i}_best.pth` — per-fold best weights
- `exohabitnet/models/checkpoints/best_model.pth` — best single-run checkpoint (if produced)
- `exohabitnet/models/checkpoints/best_ensemble.pth` — ensembled checkpoint
- `exohabitnet/reports/classification_report.md` — human-readable final metrics
- `exohabitnet/reports/model_performance.json` — machine-readable metrics
- `exohabitnet/reports/oof_probs.npy`, `oof_labels.npy` — pooled OOF probabilities and labels
- `exohabitnet/reports/oof_calibrator_*.pkl` — fitted calibrator pickles (platt/isotonic)
- `exohabitnet/reports/pipeline_diagram.svg` / `pipeline_diagram_highres.png` — architecture diagram

## Troubleshooting & tips

- If scripts error due to missing files, ensure you ran `scripts/preprocessing_pipeline.py` and that `data/processed_dataset.csv` exists.
- For reproducibility, rerun preprocessing and a small training run (e.g., `--epochs 1 --n-splits 1`) to validate environment before long experiments.
- HAB (habitable) examples are extremely rare (≈ handful of examples). Expect metrics to be unstable and focus on augmentation/sampler tuning when improving HAB detection:
  - recommended starting config used in experiments: `--use-weighted-sampler --sampler-scale 1.5 --loss focal --focal-gamma 2.0 --hab-weight-multiplier 3.0`.
